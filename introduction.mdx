Ophira Web is a fixed asset management SaaS platform that uses QR codes to track, audit, and report on physical assets across buildings and locations. It is designed for organizations that need a structured inventory of equipment, furniture, and other fixed assets assigned to specific rooms and responsible users.

Every asset in Ophira Web is identified by a unique QR code. Users can scan that code from any device with a camera to pull up the asset's full record, log a movement, or initiate an audit — no manual lookup required.

## Who it is for

Ophira Web is built for institutions and organizations that manage large inventories of physical assets across multiple buildings or campuses. Typical use cases include universities, government offices, and any facility where assets are distributed across many rooms and must be periodically audited.

## Roles

Ophira Web uses role-based access control. Every user account is assigned one of four roles, which determines what they can see and do in the platform.

| Role | Description |
|---|---|
| **Administrador** | Full access to all modules including user management, location setup, category management, and reports. Admins can create, edit, and delete any record in the system. |
| **Supervisor** | Intended for managing assets and audit workflows in an assigned area. |
| **Auditor** | Intended for conducting and recording asset audits using the QR scanner. |
| **Usuario** | Standard operational access for viewing and scanning assets. |

<Warning>
The current version of the web application only grants access to users with the **Administrador** role. Accounts assigned other roles are redirected to the login page. See [Authentication](/authentication) for full details.
</Warning>
