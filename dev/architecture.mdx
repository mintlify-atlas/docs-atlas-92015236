Ophira Web is a single-page application built with React 19 and Vite 7. All pages are client-side rendered; the backend is a separate Node.js/Express API.

## ThemeContext

`ThemeContext` provides dark/light mode state to the entire application. It is defined in `src/context/ThemeContext.jsx`.

**What it exposes:**

| Value | Type | Description |
|---|---|---|
| `isDark` | `boolean` | `true` when dark mode is active |
| `toggleTheme` | `() => void` | Flips `isDark` and persists the preference |

**Initialisation order:**

1. Reads `localStorage.getItem('theme')` on first render.
2. Falls back to `window.matchMedia('(prefers-color-scheme: dark)')` if no saved preference exists.
3. On every `isDark` change, adds or removes the `dark` class from `document.documentElement` and writes the new value to `localStorage`.

```jsx
// src/context/ThemeContext.jsx
import { createContext, useContext, useState, useEffect } from 'react';

const ThemeContext = createContext();

export const ThemeProvider = ({ children }) => {
  const [isDark, setIsDark] = useState(() => {
    const savedTheme = localStorage.getItem('theme');
    if (savedTheme) return savedTheme === 'dark';
    return window.matchMedia('(prefers-color-scheme: dark)').matches;
  });

  useEffect(() => {
    const htmlElement = document.documentElement;
    if (isDark) {
      htmlElement.classList.add('dark');
      localStorage.setItem('theme', 'dark');
    } else {
      htmlElement.classList.remove('dark');
      localStorage.setItem('theme', 'light');
    }
  }, [isDark]);

  const toggleTheme = () => setIsDark(!isDark);

  return (
    <ThemeContext.Provider value={{ isDark, toggleTheme }}>
      {children}
    </ThemeContext.Provider>
  );
};

export const useTheme = () => {
  const context = useContext(ThemeContext);
  if (!context) throw new Error('useTheme debe ser usado dentro de ThemeProvider');
  return context;
};
```

Consume the context in any component with the `useTheme` hook:

```jsx
import { useTheme } from '../context/ThemeContext';

const MyComponent = () => {
  const { isDark, toggleTheme } = useTheme();
  return (
    <button onClick={toggleTheme}>
      {isDark ? 'Switch to light' : 'Switch to dark'}
    </button>
  );
};
```

## ProtectedRoute

`src/components/protectedRoute.jsx` guards every authenticated route. It performs two checks:

1. **Token presence** — calls `getToken()`. If no token exists in `localStorage` or `sessionStorage`, redirects to `/login`.
2. **Role check** — calls `getTokenRole()`. The application only permits users with `rol === 1`. Any other role value clears the token and redirects to `/login`.

```jsx
// src/components/protectedRoute.jsx
import { Outlet, Navigate } from 'react-router-dom';
import { clearToken, getToken, getTokenRole } from '../services/authStorage';

const ProtectedRoute = ({ children }) => {
  const token = getToken();
  if (!token) {
    return <Navigate to="/login" />;
  }

  const rol = getTokenRole();
  if (rol !== 1) {
    clearToken();
    return <Navigate to="/login" />;
  }

  return <Outlet />;
};

export default ProtectedRoute;
```

<Warning>
  Only users whose JWT payload contains `rol: 1` can access the application. Users with any other role value are logged out automatically.
</Warning>

## Layout

`src/components/Layout.jsx` is the shared shell rendered around every authenticated page. It composes the fixed `Slidebar` on the left and a scrollable `<main>` content area on the right.

```jsx
// src/components/Layout.jsx
import Slidebar from './Slidebar';
import { useTheme } from '../context/ThemeContext';

const Layout = ({ children }) => {
  const { isDark } = useTheme();

  return (
    <div className={`flex min-h-screen transition-colors duration-200 ${isDark ? 'bg-slate-950' : 'bg-slate-50'}`}>
      <Slidebar />
      <main className={`flex-1 ml-64 p-8 transition-colors duration-200 ${isDark ? 'bg-slate-950' : 'bg-slate-50'}`}>
        <div className="max-w-7xl mx-auto">
          {children}
        </div>
      </main>
    </div>
  );
};
```

The sidebar is `w-64` (256 px) and fixed; the main area compensates with `ml-64`. Content is capped at `max-w-7xl` and centred.

## Sidebar navigation

`src/components/Slidebar.jsx` is a fixed left panel that includes:

- The Ophira QR logo and version label.
- `NavLink` entries for Dashboard, Activos, Leer QR, Reportes, Historial, Auditorias, and Usuarios. The active link is highlighted with `bg-blue-600 text-white`.
- A user avatar section at the bottom that fetches the current user's profile from `GET /api/usuarios/:id` (id decoded from the JWT) and navigates to `/perfil` on click.
- Dark/light mode toggle button.
- Logout button that calls `clearToken()` and navigates to `/login`.
